Chapitre 0 · Introduction

La couche
Transport

Son rôle : établir une communication entre deux applications et acheminer les données entre elles.

Suivi
Plusieurs apps simultanées

Un même hôte peut avoir plusieurs applications qui communiquent en même temps. La couche transport les suit toutes séparément.

Segmentation
Découpe les données

Les données sont divisées en segments (TCP) ou datagrammes (UDP). Plus faciles à transporter et réassembler.

Identification
Numéro de port

Chaque application est identifiée par un numéro de port. L'adresse IP = l'immeuble. Le port = l'appartement.

Protocoles
TCP et UDP

TCP/IP fournit deux protocoles : TCP pour la fiabilité, UDP pour la vitesse. Le développeur choisit selon les besoins.

Couche Transport

Les ports

Un port identifie l'application cible sur un hôte.

Port source

Généré dynamiquement par l'émetteur. Identifie la conversation.

Port destination

Informe le serveur du service demandé.

TCP

Protocole TCP

Fiable, orienté connexion. Lent mais sûr.

3 mécanismes
de fiabilité
1
Numérotation
Chaque segment reçoit un numéro d'ordre. Permet de les réorganiser s'ils arrivent dans le désordre.
2
Accusé de réception (ACK)
Le destinataire confirme chaque segment reçu.
3
Retransmission
Si aucun ACK n'arrive après un délai, le segment est renvoyé.
En bref
Orienté connexion Livraison garantie Ordre préservé Contrôle de flux En-tête ≥ 20 octets Plus lent

Le contrôle de flux = champ Fenêtre dans l'en-tête.

Applications
HTTP / HTTPS FTP (21) SMTP Telnet / SSH

TCP · Connexion

3-way
Handshake

L'établissement d'une connexion TCP se fait en 3 étapes.

1
SYN — Client → Serveur
Flag SYN + numéro d'ordre initial aléatoire x (ISN). Port destination = le service demandé (ex. 80).
2
SYN + ACK — Serveur → Client
SYN (ISN du serveur = y) + ACK (x+1). Confirme le SYN du client et annonce son propre numéro.
3
ACK — Client → Serveur
ACK = y+1. Connexion établie. Les données peuvent circuler.
Exam : Connexion = 3 étapes — SYN → SYN/ACK → ACK
TCP · Fermeture

Fermeture
de session

La fermeture se fait en 4 étapes.

1
FIN — Client → Serveur
Le client n'a plus de données. Il envoie un flag FIN.
2
ACK — Serveur → Client
Le serveur accuse réception du FIN. Session client→serveur fermée.
3
FIN — Serveur → Client
Le serveur envoie son propre FIN.
4
ACK — Client → Serveur
Le client accuse le FIN du serveur. Connexion totalement fermée.
Exam : Fermeture = 4 étapes — FIN → ACK → FIN → ACK

Commutation LAN

Adresses MAC

48 bits — identifie chaque interface réseau de façon unique.

Unicast
Un seul destinataire

Adresse unique par carte réseau. Trame envoyée d'un seul émetteur à un seul destinataire.

Broadcast
Tous les hôtes du LAN

Adresse : FF-FF-FF-FF-FF-FF (48 bits à 1). Tous les hôtes reçoivent et traitent la trame.

Multicast
Un groupe d'hôtes

Commence toujours par 01-00-5E. Plage IPv4 : 224.0.0.0 à 239.255.255.255.

Exam : Adresse commençant par 01-00-5E = multicast.
Commutation LAN

Le Switch

Équipement de couche 2. Utilise la table MAC pour acheminer intelligemment.

Table MAC
Il apprend en écoutant

À chaque trame reçue, le switch enregistre la MAC source + le port entrant dans sa table. C'est comme ça qu'il sait où se trouve chaque appareil.

Exam : Table MAC ← MAC SOURCE + PORT ENTRANT.
Décision
?
MAC destination connue
Transmet uniquement sur le port correspondant.
?
MAC destination inconnue
Flood : diffuse sur tous les ports sauf celui d'entrée.
!
Trame broadcast (FF-FF-FF-FF-FF-FF)
Toujours diffusée sur tous les ports sauf entrée.
Commutation LAN

Méthodes de
commutation

Quand le switch décide de transmettre.

1
Store-and-Forward
Reçoit la trame entière → vérifie le CRC → transmet si valide. Latence la plus élevée mais aucune erreur ne passe.
2
Cut-Through
Transmet dès que l'adresse MAC destination est lue. Pas de vérification d'erreur.
↳
Fast-Forward (variante Cut-Through)
Latence la plus faible. Transmet immédiatement après lecture de la destination.
↳
Fragment-Free (variante Cut-Through)
Attend les 64 premiers octets. Compromis entre vitesse et détection d'erreurs.
Exam : Store-and-Forward = latence MAX · Fast-Forward = latence MIN · Cut-Through = pas de CRC.
Commutation LAN

Domaines de
collision & diffusion

La notion la plus piégée de l'exam.

Collision
Où les collisions peuvent se produire

Quand deux appareils émettent en même temps sur le même segment. CSMA/CD gère ça.

Hub

Étend le domaine
3 hubs = 1 seul domaine

Switch

1 domaine par port

Diffusion
Où les broadcasts se propagent
Switch

Étend le domaine
5 switchs = 1 seul domaine

Routeur

Segmente le domaine
Ne relaie pas les broadcasts

Exam : Switch = étend la diffusion. Routeur = segmente la diffusion.
Routage

Principes
du routage

Un routeur accepte un paquet sur une interface et le transfère vers une autre.

3 étapes
1
Désencapsulation
Supprime l'en-tête et le code de fin de la trame couche 2.
2
Consultation table de routage
Examine l'adresse IP destination → cherche le meilleur chemin.
3
Réencapsulation
Encapsule dans une nouvelle trame couche 2 → transmet sur l'interface de sortie.
Règle fondamentale : L'adresse IP destination reste constante. L'adresse MAC destination change à chaque saut.

Routage Statique

Les 4 types
de routes

Chaque type répond à un besoin précis.

S
Standard — réseau spécifique
Connecte à un réseau distant. DA = 1.
ip route 192.168.2.0 255.255.255.0 10.1.1.2
S*
Par défaut — tous les paquets sans route
Masque 0.0.0.0 = aucun bit ne doit correspondre. Passerelle de dernier recours.
ip route 0.0.0.0 0.0.0.0 serial0/0/1
Σ
Récapitulative — plusieurs routes en une
Réduit la table. Condition : même interface de sortie + préfixe commun calculable.
# 172.16.8/24, .9/24, .10/24, .11/24 → 172.16.8.0/21
F
Flottante — route de secours
DA supérieure à la route principale. N'apparaît dans la table que si la principale tombe.
ip route 0.0.0.0 0.0.0.0 s0/0/1 121 # 121 > 120 (RIP) → flottante vs RIP
Exam : Flottante = DA plus élevée = inactive tant que la principale est là.
Routage Dynamique

Routage
dynamique

Les protocoles configurent automatiquement la table et s'adaptent aux changements de topologie.

Convergence
Quand tous les routeurs sont d'accord

Atteinte quand toutes les tables sont cohérentes et complètes. Le réseau n'est pas opérationnel avant ça.

RIP / IGRP

Convergence lente

EIGRP / OSPF

Convergence rapide

Facteur principal : taille du réseau.

2 familles
Vecteur de distance
Ne connaît pas le chemin complet.
Utilise ses voisins comme repères.
exemplesRIP, IGRP, EIGRP
État de liens
Vue complète de la topologie.
Chaque routeur calcule ses propres chemins.
exemplesOSPF, IS-IS
IGP vs EGP
IGP

Au sein d'un seul système autonome. RIP, EIGRP, OSPF, IS-IS.

EGP

Entre systèmes autonomes différents. BGP pour Internet.
