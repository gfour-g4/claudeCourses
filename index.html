La Source Discrète
Définitions essentielles
Def Source discrète

Émet des symboles appartenant à un alphabet fini A = {s₀, s₁, …, sₖ₋₁}

Def Source aléatoire

Chaque symbole sₖ est émis avec probabilité P(S=sₖ) = pₖ, avec Σpₖ = 1

Def Source sans mémoire

Le symbole émis à l'instant k est indépendant des symboles précédemment émis

💡 La quantité d'information d'un symbole est inversement proportionnelle à sa probabilité. Un événement rare = beaucoup d'information (surprise). Un événement certain = zéro information.

Quantité d'Information & Entropie
Formules clés à maîtriser
Quantité d'information d'un symbole
I(sₖ) = -log₂(pₖ) [bits] • I(sₖ) = 0 si pₖ = 1 (aucune surprise) • I(sₖ) ≥ 0 pour 0 ≤ pₖ ≤ 1 • I(sₖ) > I(sᵢ) si pₖ < pᵢ (plus rare = plus d'info) • I(sₖsᵢ) = I(sₖ) + I(sᵢ) si indépendants
Entropie d'une source (info moyenne / symbole)
H(S) = -Σ pₖ · log₂(pₖ) = Σ pₖ · log₂(1/pₖ) [bits/symbole] Bornes : 0 ≤ H(S) ≤ log₂(k) H(S) = 0 → source certaine (1 seul symbole) H(S) = log₂(k) → symboles équiprobables (max!)
🎲 Exemple : Source binaire

P(1) = 0.3, P(0) = 0.7

H(S) = -0.7·log₂(0.7) - 0.3·log₂(0.3) = 0.7×0.515 + 0.3×1.737 ≈ 0.88 bits/symbole
🎲 Exemple : Dé truqué

P(1)=P(6)=0.2 ; P(2)=P(3)=P(4)=P(5)=0.15

H(S) = 2×[-0.2·log₂(0.2)] + 4×[-0.15·log₂(0.15)] ≈ 2.571 bits/symbole Dé non truqué : H = log₂(6) ≈ 2.585 sh → L'entropie est max si équiprobable !
Propriété clé : L'entropie est maximale quand les symboles sont équiprobables. Plus l'incertitude est grande, plus l'entropie est élevée. Elle ne dépend pas des symboles eux-mêmes, mais de leurs probabilités.

Canal de Transmission
Capacité, information mutuelle, canaux classiques
Def Canal Binaire Symétrique (CBS)
  • Xₖ, Yₖ ∈ {0,1}
  • p = probabilité d'erreur par bit
  • 1−p = probabilité de transmission correcte
  • Les 0 et 1 sont affectés de la même façon
Capacité CBS : C = 1 + p·log₂(p) + (1-p)·log₂(1-p) • p=0 → C=1 (sans bruit, parfait) • p=0.5 → C=0 (canal inutile!) • p=1 → C=1 (inversion parfaite → corrigeable)
Def Information Mutuelle
I(A,B) = H(A) - H(A|B) = H(B) - H(B|A) = H(A) + H(B) - H(A,B) • I(A,B) ≥ 0 • Nul si A et B indépendants • H(A|B) = "information perdue" (bruit)
Def Capacité d'un canal
C = max_{p(xᵢ)} I(X,Y) = max { H(X) - H(X|Y) } Débit max = C/Tc [bits/s] (Tc = durée par symbole)
Théorème de Shannon : Il existe un code permettant de transmettre à la capacité du canal si le débit d'information de la source vérifie H(S) × Ds ≤ C
🔌 Canal avec Effacement (CBE)
Entrées ∈ {0,1}, sorties ∈ {0, e, 1}.
e = symbole effacé (perdu) avec probabilité α.
La transmission correcte se fait avec probabilité 1−α.

Codage Source — Concepts fondamentaux
Codes préfixes, Kraft, efficacité
PropriétéDéfinition
RégulierDeux symboles distincts → deux mots de code distincts (injectif)
DéchiffrableToute suite de mots de code a une unique interprétation
PréfixeAucun mot de code n'est le début d'un autre (condition suffisante)
InstantanéDécodable au fur et à mesure (⟺ préfixe)
CompletIntelligible + tout ajout le rend non intelligible. Lp=0 pour p > n
Longueur moyenne d'un code
|φ| = Σ |φ(x)| · pₓ [bits/symbole] Efficacité : E(φ) = H / |φ| (≤ 1 toujours) Longueur min théorique : l_min = H(S) / log₂(V) Rendement : R = l_min / l_moy Redondance : ρ = 1 - R
Inégalité de Kraft (condition nécessaire & suffisante)
Σᵢ q^(-nᵢ) ≤ 1 q = valence (2 pour binaire) nᵢ = longueur du i-ème mot de code → Code préfixe existe ssi cette inégalité est satisfaite
Attention : Être préfixe est une condition suffisante mais non nécessaire pour la décodabilité unique. Un code peut être déchiffrable sans être préfixe (mais c'est rare).

Codage de Huffman
Shannon-Fano, Huffman classique & variance minimale

📊 Algorithme Shannon-Fano

1

Ranger les symboles par probabilité décroissante

2

Diviser en 2 groupes G₀ et G₁ de probabilités les plus voisines (sans changer l'ordre)

3

Subdiviser récursivement chaque groupe Gᵢ → Gᵢ₀ et Gᵢ₁

4

S'arrêter quand chaque groupe = 1 seul élément. L'indice du groupe = le mot de code

🌲 Algorithme Huffman

1

Construire un arbre en partant des feuilles les plus profondes (états de la source)

2

À chaque étape, fusionner les 2 nœuds de plus faible probabilité en un nœud somme

3

S'arrêter quand il ne reste qu'une seule feuille (la racine)

4

Le parcours inverse de l'arbre fournit les mots de code (0 = branche haute, 1 = basse)

🎯 Exemple Huffman — Source à 5 symboles
SymboleProbabilitéCode Huffman
s₁0.3500
s₂0.2201
s₃0.1810
s₄0.15110
s₅0.10111
φ = 0.35×2 + 0.22×2 + 0.18×2 + 0.15×3 + 0.10×3 = 0.70 + 0.44 + 0.36 + 0.45 + 0.30 = 2.25 bits/symbole H(S) = 2.19 bits/symbole Efficacité = 2.19/2.25 ≈ 0.973 → très bon !
Propriété : Huffman fournit un code instantané de longueur moyenne minimale. Les deux symboles les moins probables ont les codes les plus longs et ne diffèrent que d'un bit.
⚡ Variance Minimale — Pourquoi c'est important
  • Plusieurs arbres de Huffman peuvent exister avec la même longueur moyenne
  • Le codeur à variance minimale a le décodage le plus rapide (temps réel)
  • Mots de code de longueurs proches → lecture en blocs → plus rapide
Var = Σ pᵢ · (lᵢ - |φ|)² Règle : en cas d'égalité de probabilité, placer l'ensemble avec le plus de lettres en haut (à droite dans l'arbre)

Codage Arithmétique
Statique & Adaptatif — JPEG, JBIG
Principe : Associer à toute une chaîne de symboles un nombre réel dans [0,1[. Permet de coder sur un nombre non entier de bits par symbole → plus efficace que Huffman.
📐 Procédure de codage statique
1

Diviser [0,1[ en sous-intervalles proportionnels aux probabilités des symboles

2

Pour chaque symbole de la chaîne, choisir son sous-intervalle comme nouvel intervalle de travail

3

À la fin, choisir un nombre réel dans l'intervalle final → le convertir en binaire

4

Précision = nombre de chiffres après la virgule = longueur de la chaîne

📊 Exemple : coder "bbac"

a:0.7, b:0.1, c:0.2

Itération 1 (b): intervalle = [0.7, 0.8[ Itération 2 (b): intervalle = [0.77, 0.78[ Itération 3 (a): intervalle = [0.770, 0.777[ Itération 4 (c): intervalle = [0.7756, 0.7770[ → Code : 0.7762 (appartient à l'intervalle)
🔄 Décodage arithmétique
Reçu: 0.7762 0.7762 ∈ [0.7, 0.8[ → b, soustraire 0.7, ÷0.1 0.762 ∈ [0.7, 0.8[ → b, soustraire 0.7, ÷0.1 0.62 ∈ [0.0, 0.7[ → a, soustraire 0.0, ÷0.7 0.88 ∈ [0.8, 1.0[ → c ✓ "bbac"
🔄 Codage Arithmétique Adaptatif
  • Pas de connaissance a priori des probabilités
  • Initialisation : tous les symboles ont le même compteur = 1 (équiprobable)
  • Après chaque symbole codé, incrémenter son compteur → mettre à jour les probabilités
  • Le décodeur fait exactement la même mise à jour après chaque décodage
  • Aucune transmission des probabilités en en-tête nécessaire
  • Utilisé dans JPEG2000, MPEG4, JBIG
Huffman vs Arithmétique : Arithmétique est légèrement + efficace (pas d'entier de bits par symbole) mais + lourd en calcul. Huffman reste le standard pour les applications temps réel.

RLE & Codage à Dictionnaire (LZW)
Redondance spatiale & temporelle

🔁 RLE — Run-Length Encoding

  • Exploite les séquences de symboles répétés
  • Remplace une suite de n répétitions par (n, symbole)
  • Efficace pour images N&B, fax, images médicales
  • Inefficace si peu de répétitions
Exemple : "HellooooooooWoooooooorld" → 1H 1e 2l 8o 1W 8o 1r 1l 1d → 24 car. → 18 car. = -25% ✓ "HelloWorld" → 1H 1e 2l 1o 1W 1o 1r 1l 1d → 10 car. → 18 car. = +80% ✗ (contre-productif)

📚 LZW — Codage à Dictionnaire

  • Construit un dictionnaire dynamique des séquences vues
  • Encode les chaînes récurrentes par leurs indices dans le dictionnaire
  • Adaptatif — pas besoin de transmettre le dictionnaire
  • Utilisé dans GIF, ZIP, PDF, PNG
Principe : 1. Lire le flux d'entrée caractère par caractère 2. Si la chaîne courante est dans le dico → continuer 3. Sinon → émettre l'index + ajouter la chaîne au dico 4. Le décodeur reconstruit le même dico en parallèle
📠 Codage Huffman Modifié — FAX
  • Norme ITU depuis 1993 pour les machines FAX
  • Combine RLE + Huffman sur images N&B
  • Ligne de balayage → séquences de pixels blancs/noirs
  • Codes terminaux (1-63 pixels) + codes de mise en page (multiples de 64)
Exemple : WWWWWWWWWWWWBWWWWWWWWWWWWBBBW… → RLE : 12W 1B 12W 3B 24W 1B 14W → Huffman sur les longueurs de runs → Compression 97× pour ligne noire!

Taux de compression τ = Vol(Ĩ) / Vol(I) Quotient q = Vol(I) / Vol(Ĩ) = 1/τ MSE = (1/3MN) ΣΣΣ (I - Î)² PSNR = 10·log₁₀(d²/MSE) [dB] (d = valeur max pixel, ex: 255)

Σ (Lp / V^(p-1)) = V p de 1 à n Lp = nombre de mots de code de longueur p V = valence du code (binaire V=2) → Condition pour code complet intelligible

📘 Exercice 1 — Entropie & Codage de Huffman
Soit une source S à 6 symboles avec les probabilités suivantes :
s₁=0.30, s₂=0.25, s₃=0.20, s₄=0.12, s₅=0.08, s₆=0.05
  1. Calculer l'entropie H(S) de la source.
  2. Construire le code de Huffman binaire et donner les mots de code.
  3. Calculer la longueur moyenne |φ| et l'efficacité E du code obtenu.
  4. Vérifier l'inégalité de Kraft.
📗 Voir la solution complète
1. Calcul de l'entropie H(S)
H(S) = -Σ pᵢ·log₂(pᵢ) = -0.30·log₂(0.30) - 0.25·log₂(0.25) - 0.20·log₂(0.20) - 0.12·log₂(0.12) - 0.08·log₂(0.08) - 0.05·log₂(0.05) = 0.30×1.737 + 0.25×2.000 + 0.20×2.322 + 0.12×3.059 + 0.08×3.644 + 0.05×4.322 = 0.521 + 0.500 + 0.464 + 0.367 + 0.292 + 0.216 H(S) ≈ 2.360 bits/symbole Borne max : log₂(6) ≈ 2.585 bits → source non équiprobable
2. Construction de l'arbre de Huffman
ÉtapeActionRésultat
1Fusionner s₅(0.08) + s₆(0.05)N₁ = 0.13
2Fusionner N₁(0.13) + s₄(0.12)N₂ = 0.25
3Fusionner s₃(0.20) + N₂(0.25)N₃ = 0.45
4Fusionner s₂(0.25) + s₁(0.30)N₄ = 0.55
5Fusionner N₃(0.45) + N₄(0.55)Racine = 1.00
Mots de code (branche gauche=0, droite=1) : s₁ = 0.30 → 10 s₂ = 0.25 → 11 s₃ = 0.20 → 00 s₄ = 0.12 → 011 s₅ = 0.08 → 0100 s₆ = 0.05 → 0101
3. Longueur moyenne et efficacité
|φ| = 0.30×2 + 0.25×2 + 0.20×2 + 0.12×3 + 0.08×4 + 0.05×4 = 0.60 + 0.50 + 0.40 + 0.36 + 0.32 + 0.20 = 2.38 bits/symbole Efficacité : E = H(S) / |φ| = 2.360 / 2.38 ≈ 0.992 ✓ (très bon !)
4. Vérification de Kraft
Σ 2^(-nᵢ) = 2×2⁻² + 1×2⁻³ + 2×2⁻⁴ = 2×0.25 + 0.125 + 2×0.0625 = 0.50 + 0.125 + 0.125 + 0.0625 + 0.0625 ... (détail par symbole) = 2⁻² + 2⁻² + 2⁻² + 2⁻³ + 2⁻⁴ + 2⁻⁴ = 0.25 + 0.25 + 0.25 + 0.125 + 0.0625 + 0.0625 = 1.0 ≤ 1 ✓
Kraft = 1 → code complet, aucun bit gaspillé ✓

📗 Exercice 3 — Shannon-Fano & Comparaison
Source S = {A, B, C, D, E} avec P(A)=0.4, P(B)=0.2, P(C)=0.2, P(D)=0.1, P(E)=0.1
  1. Construire le code Shannon-Fano.
  2. Construire le code Huffman.
  3. Comparer les longueurs moyennes et les efficacités.
  4. Calculer H(S) et conclure sur l'optimalité.
📗 Voir la solution complète
4. Entropie H(S)
H(S) = -2×[0.4·log₂(0.4)] - 2×[0.2·log₂(0.2)] - 2×[0.1·log₂(0.1)] = 0.4×1.322 + 2×(0.2×2.322) + 2×(0.1×3.322) = 0.529 + 0.929 + 0.664 H(S) ≈ 2.122 bits/symbole
1. Shannon-Fano
Tri décroissant : A(0.4) B(0.2) C(0.2) D(0.1) E(0.1) Total = 1.0 Division 1 : G₀={A, B} (0.6) vs G₁={C, D, E} (0.4) G₀ → bit 0 : A(0.4) | B(0.2) Sous-division : G₀₀={A} (0.4) | G₀₁={B} (0.2) → A = 00, B = 01 G₁ → bit 1 : C(0.2) | D(0.1) | E(0.1) Sous-division : G₁₀={C} (0.2) | G₁₁={D, E} (0.2) → C = 10 G₁₁={D, E} : G₁₁₀={D} | G₁₁₁={E} → D = 110, E = 111 Codes Shannon-Fano : A=00, B=01, C=10, D=110, E=111
2. Huffman
Tri : A(0.4) B(0.2) C(0.2) D(0.1) E(0.1) Étape 1 : D(0.1) + E(0.1) → N₁(0.2) File : A(0.4) B(0.2) C(0.2) N₁(0.2) Étape 2 : B(0.2) + C(0.2) → N₂(0.4) [ou C + N₁] File : A(0.4) N₂(0.4) N₁(0.2) Étape 3 : N₁(0.2) + N₂(0.4) → N₃(0.6) File : A(0.4) N₃(0.6) Étape 4 : A(0.4) + N₃(0.6) → Racine(1.0) Codes Huffman : A=0, B=110, C=111, D=100, E=101 (ou variante : A=0, B=10, C=110, D=1110, E=1111 selon arbre)
3. Comparaison
CodeA(0.4)B(0.2)C(0.2)D(0.1)E(0.1)|φ|E
Shannon-Fano00 (2)01 (2)10 (2)110 (3)111 (3) 2.20.965
Huffman0 (1)110 (3)111 (3)100 (3)101 (3) 2.20.965
Conclusion : Les deux codes donnent ici |φ| = 2.2 bits/symbole, soit la même efficacité (0.965). Huffman garantit toujours la longueur moyenne minimale. Les deux sont quasi-optimaux par rapport à H(S)=2.122.
📕 Exercice 4 — Canal Binaire Symétrique & Théorème de Shannon
Un CBS a une probabilité d'erreur p = 0.1. La source émet 1200 symboles/seconde avec H(S) = 0.8 bits/symbole.
  1. Calculer la capacité C du canal (en bits par usage).
  2. Vérifier si la transmission est possible selon Shannon.
  3. Quel débit maximal (symboles/seconde) peut-on transmettre si la durée par symbole Tc = 1ms ?
  4. Calculer H(X|Y) (équivoque du canal).
📗 Voir la solution complète
1. Capacité du CBS
C = 1 + p·log₂(p) + (1-p)·log₂(1-p) = 1 + 0.1·log₂(0.1) + 0.9·log₂(0.9) = 1 + 0.1×(-3.322) + 0.9×(-0.152) = 1 - 0.332 - 0.137 C ≈ 0.531 bits/usage
2. Vérification de Shannon
Condition : H(S) × Ds ≤ C Débit d'information source : Ds = H(S) × (symboles/s) = 0.8 × 1200 = 960 bits/s Capacité canal (bits/s) : C × (1/Tc) = 0.531 × 1000 = 531 bits/s 960 bits/s > 531 bits/s → ❌ La transmission fiable est IMPOSSIBLE à ce débit ! → Il faut réduire le débit source ou utiliser un canal de plus grande capacité.
3. Débit maximal admissible
On cherche Ds_max tel que H(S)×Ds_max ≤ C/Tc Ds_max = C/(Tc × H(S)) = 0.531 / (0.001 × 0.8) = 0.531 / 0.0008 ≈ 664 symboles/s → La source doit émettre au plus 664 symboles/seconde pour que Shannon soit respecté.
4. Équivoque H(X|Y)
H(X|Y) = H(Y|X) pour un CBS (symétrique) = -p·log₂(p) - (1-p)·log₂(1-p) = -0.1·log₂(0.1) - 0.9·log₂(0.9) = 0.332 + 0.137 H(X|Y) ≈ 0.469 bits/usage Vérification : C = H(X) - H(X|Y) = 1 - 0.469 = 0.531 ✓ (source équiprobable)
📘 Exercice 5 — Kraft, Codes et Knat-McMillan
Soit les codes binaires suivants pour une source à 5 symboles :
Code A : {0, 10, 110, 1110, 1111}
Code B : {0, 1, 00, 01, 10}
Code C : {0, 10, 110, 1110, 11110}
  1. Pour chaque code, vérifier l'inégalité de Kraft. Est-il préfixe ? Complet ?
  2. Lequel est optimal au sens de Huffman ?
  3. Appliquer Knat-McMillan à Code A.
📗 Voir la solution complète
1. Vérification de Kraft
Code A : longueurs = 1, 2, 3, 4, 4 ΣKraft = 2⁻¹ + 2⁻² + 2⁻³ + 2⁻⁴ + 2⁻⁴ = 0.5 + 0.25 + 0.125 + 0.0625 + 0.0625 = 1.0 ≤ 1 → Kraft = 1 : Code préfixe ET complet ✓ Code B : longueurs = 1, 1, 2, 2, 2 → "0" est préfixe de "00" et "01" ! → PAS déchiffrable ΣKraft = 2⁻¹ + 2⁻¹ + 2⁻² + 2⁻² + 2⁻² = 0.5 + 0.5 + 0.75 = 1.75 > 1 → Kraft > 1 : Code NON préfixe, NON déchiffrable ✗ Code C : longueurs = 1, 2, 3, 4, 5 ΣKraft = 2⁻¹ + 2⁻² + 2⁻³ + 2⁻⁴ + 2⁻⁵ = 0.5 + 0.25 + 0.125 + 0.0625 + 0.03125 = 0.969 < 1 → Kraft < 1 : Code préfixe mais PAS complet (redondant) ⚠
2. Code optimal de Huffman
Code A est le seul candidat valide (Kraft=1, préfixe, complet). C'est la structure typique du code de Huffman pour une source dont les probabilités décroissent rapidement (ex: géométrique). Code C est gaspilleur. Code B est invalide.
3. Knat-McMillan sur Code A
Knat-McMillan : Σ (Lp / V^(p-1)) = V (pour code complet, V=2) Lp = nombre de mots de code de longueur p dans le code : L₁ = 1 (le mot "0") L₂ = 1 (le mot "10") L₃ = 1 (le mot "110") L₄ = 2 (les mots "1110" et "1111") Vérification : L₁/V⁰ + L₂/V¹ + L₃/V² + L₄/V³ = 1/1 + 1/2 + 1/4 + 2/8 = 1.0 + 0.5 + 0.25 + 0.25 = 2.0 = V ✓ → Code complet intelligible confirmé
📗 Exercice 6 — RLE, Taux de Compression & PSNR
Une image N&B de 100×8 pixels est représentée comme suit (W=blanc, N=noir) :
Chaque ligne contient : WWWWWWWWNWWWWWWWWNNNW (répétée sur toutes les lignes, puis padding à 100)
  1. Encoder la ligne donnée avec RLE.
  2. Calculer le taux de compression τ (avant vs après RLE) pour 1 ligne de 100 pixels.
  3. Une image 8 bits 256×256 a un MSE = 2.5 après compression. Calculer le PSNR.
📗 Voir la solution complète
1. Encodage RLE
Séquence : WWWWWWWWNWWWWWWWWNNNW (21 pixels) + padding jusqu'à 100 pixels = on suppose 79W supplémentaires Séquence complète : WWWWWWWWNWWWWWWWWNNNWWWWWWWWWWWWWWWWWWWWWWWWWWWWWWWWWWWWWWWWWWWWWWWWWWWWWWWWWWWW (100W) RLE : 8W → (8, W) 1N → (1, N) 8W → (8, W) 3N → (3, N) 80W → (80, W) [20-21 → 79+1 = 80W] RLE = 5 paires (10 valeurs) au lieu de 100 pixels → Compression : 100 pixels → 10 valeurs = taux 10:1
2. Taux de compression τ
Volume original : 100 pixels × 1 bit = 100 bits (image N&B) Volume compressé : 5 paires (n, symbole) n ∈ [1, 100] → besoin de 7 bits pour n symbole → 1 bit (0=W, 1=N) → 5 × (7+1) = 40 bits τ = Vol(compressé) / Vol(original) = 40 / 100 = 0.40 Quotient q = 1/τ = 2.5 → compression 2.5:1
3. Calcul du PSNR
Image 8 bits : d = 2⁸ - 1 = 255 (valeur max pixel) MSE = 2.5 PSNR = 10·log₁₀(d²/MSE) = 10·log₁₀(255²/2.5) = 10·log₁₀(65025/2.5) = 10·log₁₀(26010) = 10 × 4.415 PSNR ≈ 44.15 dB ✓ Interprétation : PSNR > 40 dB → excellente qualité visuelle (quasi-transparent)
